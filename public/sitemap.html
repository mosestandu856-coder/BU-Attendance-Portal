Current System Features

Authentication

  • User login with credentials
  • Role-based access control
  • WebAuthn fingerprint support
  • Account self-registration
  • First user becomes Admin

Reporting & Analytics

  • Per-course attendance stats
  • Lecturer performance reports
  • At-risk student detection (<75%)
  • Print preview with BU letterhead
  • QA monitoring dashboard

Contact & Support

  • Students/Lecturers/QA send messages
  • Admin views all messages
  • Admin replies via email client
  • Message status tracking

Attendance Tracking

  • QR code generation & scanning
  • Real-time attendance recording
  • GPS location verification
  • Biometric fingerprint verification
  • Course-based attendance filter
  • QR expiry settings

User Roles

  • Admin – full system control
  • Lecturer – manage courses & lectures
  • Student – enroll, scan & view records
  • QA Office – monitor & report

Course Management

  • Admin creates & assigns courses
  • Student enrollment requests
  • Admin approval workflow
  • Semester management
  • Lecturer-course assignment
  • Course-specific lecture creation

Technology Stack

  • Node.js + Express backend
  • SQLite (sql.js) database
  • HTML5 QR Scanner
  • Leaflet Maps integration
  • WebAuthn biometric API
  • Chart.js analytics
  • Nodemailer (contact replies)
